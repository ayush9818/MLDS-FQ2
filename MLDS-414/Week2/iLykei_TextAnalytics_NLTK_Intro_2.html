Lexical resources

Lexicon or ** lexical resource** in NLP means a collection of words/phrases along with some associated information, such as part-of-speech (pos) sense definition, semantics, synonyms, etc.

Lexicon consists of rows called lexical entries. A typical structure of lexical entry is:

  • Headword, also known as lemma. This is the reference word
  • Part-of-speech or lexical category
  • Sense definition or gloss

Simplest lexicon is just a sorted list of words.

Lexicons contain the most important information for natural language processing: they allow analyzing the meaning of the text as opposed to collecting some summary statistics like frequencies of words, etc.

Some of the most important lexical resources contained in NLTK are reviewed below.

Corpus words

Corpus called words is the /usr/dict/words file from Unix. It can be used to find unusual or misspelled words contained in the document.

The following function creates vocabulary of the text, then removes all "standard" words contained in a wordlist leaving only unusual or misspelled words.

Function s.isalpha() tests if s is non-empty and all characters in s are alphabetic.

In[2]:
def unusual_words(text):
    text_vocab=set(w.lower() for w in text if w.isalpha())
    english_vocab=set(w.lower() for w in nltk.corpus.words.words())
    unusual=text_vocab.difference(english_vocab)
    return sorted(unusual)

In[6]:
def content_fraction(text):
    stopwords=nltk.corpus.stopwords.words('english')
    content=[w for w in text if w.lower() not in stopwords]
    return len(content)/len(text)

Create a function that solves the following well known word puzzle:

Create as many as possible words of 4 letters or more from the given letters 'egivrvonl'. Each letter can be used only once per word. Each word must contain letter 'r' and there must be at least one 9-letter word. No plurals ending in "s", no foreign words, no proper names.

Results:

  • 21 words: good;
  • 32 words: very good;
  • 42 words: excellent.
In[8]:
def solve_puzzle(given_letters,obligatory):
    puzzle_letters=nltk.FreqDist(given_letters)
    wordlist=nltk.corpus.words.words()
    return [w for w in wordlist if \
            len(w) >= 4 \
           and obligatory in w \
           and nltk.FreqDist(w) <= puzzle_letters]

WordNet

It is important to have a measure of distance between words based on the similarity of their meaning or semantic content as opposed to similarity of their syntactical representation (e.g. their string format or word count).

WordNet created in Cognitive Science Laboratory of Princeton University gives tools for measuring semantic similarity. Wordnet is a semantically oriented English dictionary, similar to thesaurus.

Nouns, verbs, adjectives and adverbs in this database are grouped into cognitive synonym sets (synsets).

The most frequently encoded relation among synsets is the super-subordinate relation (also called hyperonymy, hyponymy or ISA relation).

WordNet links more general synsets like furniture, piece_of_furniture to increasingly specific ones like bed and bunkbed.

See more detailed description on the WordNet page or here.

Hypernyms and hyponyms are lexical relations, they relate synsets. Other types of relations are meronyms and holonyms.

Parts of a tree are meronyms:

  • trunk, leaf, etc, are part_meronyms()
  • A substance a tree is made of, like heartwood or sapwood are substance_meronyms()

What is composed, formed with trees - forest - is a member_holonyms()

Function synset1.path_similarity(synset2) returns a score denoting how similar two word senses are, based on the shortest path that connects the senses in the is-a (hypernym/hyponym) taxonomy. The score is in the range 0 to 1. Large score means similar words.

In[30]:
dog = wn.synset('dog.n.01')
pet = wn.synset('pet.n.01')
table = wn.synset('table.n.01')
print('Similarity of dog to pet = %s' % dog.path_similarity(pet))
print('Similarity of dog to table = %s' % dog.path_similarity(table))
print('Similarity of dog to horse = %s' % \
      wn.synsets('dog')[0].path_similarity(wn.synsets('horse')[0]))
