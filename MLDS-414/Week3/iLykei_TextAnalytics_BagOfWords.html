The approach based on the so called bag-of-words models is the simplest method in NLP. It does not require any deep understanding of the language structure and is based on relative frequencies of words from certain vocabulary in the text.

A bag-of-words is a representation of text that describes the occurrence of words within a document and is based on two things:

  • A vocabulary of known words.
  • A measure of the presence of known words.

It is called a bag-of-words , because any information about the order or structure of words in the document is discarded. The model is only concerned with whether known words occur in the document, not where in the document.

Vocabulary is the key element of the model. It can be created using the given document or corpus of documents. Alternatively it can be one of standard vocabularies based on a large number of documents.

The Movie Review Data¶

The Movie Review Data is a collection of movie reviews retrieved from the imdb.com website in the early 2000s by Bo Pang and Lillian Lee.

The reviews collected and made available as part of their research on natural language processing, were originally released in 2002. An updated and cleaned version was released in 2004, referred to as v2.0. The dataset contains 1,000 positive and 1,000 negative movie reviews drawn from an archive of the rec.arts.movies.reviews newsgroup hosted at IMDB. The reviews were written before 2002, with a cap of 20 reviews per author (312 authors total) per category.
The authors refer to this dataset as the polarity dataset.

Some data preparation has already been done in the set. For example:

  • Only English reviews selected
  • Texts converted to lowercase
  • White space added around punctuation like periods, commas, and brackets
  • Texts split into one sentence per line

Download the Movie Review Polarity Dataset from here.

The archive with the data contains directory called txt_sentoken with two sub-directories in it called neg and pos for negative and positive reviews.
Reviews are stored one per file with a naming convention from cv000 to cv999 for each of neg and pos.

# turn a doc into clean tokens
def clean_doc(doc):
    # split into tokens by white space
    tokens = doc.split()
    # prepare regex for char filtering
    re_punc = re.compile('[%s]' % re.escape(string.punctuation))
    # remove punctuation from each word
    tokens = [re_punc.sub('', w) for w in tokens]
    # remove remaining tokens that are not alphabetic
    tokens = [word for word in tokens if word.isalpha()]
    # filter out stop words
    stop_words = set(stopwords.words('english'))
    tokens = [w for w in tokens if not w in stop_words]
    # filter out short tokens
    tokens = [word for word in tokens if len(word) > 1]
    return tokens

Creating Vocabulary¶

Size of vocabulary plays very important role. If it is not large enough many meaningful words may be missed in the analysis. On the other hand, if vocabulary is too large the words representation will be too sparse.
If the vocabulary is created from the given corpus of documents load all the documents and then analyze words that need to be dropped or kept depending on the task.

Working with the vocabulary is convenient using a Counter, which is a dictionary of words and their counts with some additional functions.
Create a new function processing a document and adding the words from it to the vocabulary.
The function:

  • Loads the document by calling load_doc()
  • Cleans the loaded document using clean_doc()
  • Adds all the tokens to the Counter and updates counts by calling the update() function on the counter object.

Below is a function called add_doc_to_vocab() that takes as arguments a document filename and a Counter vocabulary.

from collections import Counter

# load doc and add to vocab
def add_doc_to_vocab(filename, vocab):
    # load doc
    doc = load_doc(filename)
    # clean doc
    tokens = clean_doc(doc)
    # update counts
    vocab.update(tokens)

# load doc, clean and return line of tokens
def doc_to_line(filename, vocab):
    # load the doc
    doc = load_doc(filename)
    # clean doc
    tokens = clean_doc(doc)
    # filter by vocab
    tokens = [w for w in tokens if w in vocab]
    return ' '.join(tokens)

Comparing word scoring methods¶

The function text_to_matrix() gives 4 scoring options:

  • binary: words are scored as present (1) or absent
  • count: integer count for each word
  • tfidf: each word is scored based on their frequency, where words that are common across all documents are penalized
  • freq: words are scored based on their frequencies.

To compare the scoring options create a function prepare_data() encoding the texts with the selected encoding mode.
It creates the tokenizer, fits it on the training documents, then creates the train and test encodings using the chosen mode and given lists of train and test documents.

In [35]:
def prepare_data(train_docs, test_docs, mode):
    # create the tokenizer
    tokenizer = Tokenizer()
    # fit the tokenizer on the documents
    tokenizer.fit_on_texts(train_docs)
    # encode training data set
    Xtrain = tokenizer.texts_to_matrix(train_docs, mode=mode)
    # encode training data set
    Xtest = tokenizer.texts_to_matrix(test_docs, mode=mode)
    return Xtrain, Xtest

In addition create function evaluate_mode() for evaluation of different models. It runs each model several times for more reliable evaluation.

# evaluate a neural network model
def evaluate_mode(Xtrain, ytrain, Xtest, ytest):
    scores = list()
    n_repeats = 10
    n_words = Xtest.shape[1]
    for i in range(n_repeats):
        # define network
        model = define_model(n_words)
        # fit network
        model.fit(Xtrain, ytrain, epochs=10, verbose=0)
        # evaluate
        _, acc = model.evaluate(Xtest, ytest, verbose=0)
        scores.append(acc)
        print('%d accuracy: %s' % ((i+1), acc))
    return scores

def load_clean_dataset(vocab):
    # load documents
    neg = process_docs(dataPath/'txt_sentoken/neg', vocab)
    pos = process_docs(dataPath/'txt_sentoken/pos', vocab)
    docs = neg + pos
    # prepare labels
    labels = array([0 for _ in range(len(neg))] + [1 for _ in range(len(pos))])
    return docs, labels

Create predict_sentiment() which takes review, vocabulary, tokenizer and the trained model and returns the predicted sentiment.

In [41]:
# classify a review as negative or positive
def predict_sentiment(review, vocab, tokenizer, model):
    # clean
    tokens = clean_doc(review)
    # filter by vocab
    tokens = [w for w in tokens if w in vocab]
    # convert to line
    line = ' '.join(tokens)
    # encode
    encoded = tokenizer.texts_to_matrix([line], mode='binary')
    # predict sentiment
    yhat = model.predict(encoded, verbose=0)
    # retrieve predicted percentage and label
    percent_pos = yhat[0,0]
    if round(percent_pos) == 0:
        return (1-percent_pos), 'NEGATIVE'
    return percent_pos, 'POSITIVE'
