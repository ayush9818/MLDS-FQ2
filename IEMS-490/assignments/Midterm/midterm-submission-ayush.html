Question 1¶

Are the recurrent architecture (RNN with only one layer) and the attention architecture (transformer with only one layer) invariant/equivariant to permutations of inputs, i.e., giving the same set of outputs (reordered) upon reordering the input sequence

Answer 1¶

1. Recurrent Architecture (RNN with only one layer)¶

For a single-layer RNN, the hidden state at each time step depends on both the current input and the hidden state from the previous time step. This means that RNNs inherently process sequences in an ordered fashion.

  • Invariance/Equivariance: A single-layer RNN is neither invariant nor equivariant to input permutations. If you reorder the input sequence, the RNN's outputs will likely change completely because each element's position in the sequence influences how the hidden state is updated. Changing the order of inputs disrupts this dependency chain, so there’s no straightforward relationship between the original output and the output of the reordered sequence.

2. Attention Architecture (Transformer with only one layer)¶

Transformers, particularly self-attention layers, calculate pairwise dependencies between all elements in a sequence without explicit sequential processing, which suggests they may handle permutations differently from RNNs.

Case A: Transformer Without Positional Encoding¶

Without positional encoding, the self-attention mechanism has no information about the order of inputs; it treats all positions as indistinguishable.

  • Invariance/Equivariance: A Transformer without positional encoding is equivariant to permutations of inputs. Since there is no positional information, reordering the input will reorder the output in the same way. For example, if the input tokens are shuffled, the output will reflect this shuffling without changing the content, essentially "following" the permutation applied to the input.

Case B: Transformer With Positional Encoding¶

With positional encoding, each input position has a unique embedding that represents its relative or absolute position in the sequence. This positional information is added to the input embeddings before they are fed to the attention layers, which allows the model to distinguish among different positions in the sequence.

  • Invariance/Equivariance: A Transformer with positional encoding is neither invariant nor equivariant to input permutations. The positional encoding ties each input to its specific position, so reordering inputs leads to outputs that are not directly aligned with the permutation of the inputs. The change in input ordering disrupts the positional encoding and the resulting attention patterns, which in turn affects the output structure in a non-permutative manner.

class TimeSeriesDataset(Dataset):
    def __init__(self, data, seq_len):
        self.data = torch.tensor(data, dtype=torch.float32)
        self.seq_len = seq_len

    def __len__(self):
        return len(self.data) - self.seq_len

    def __getitem__(self, idx):
        x = self.data[idx:idx + self.seq_len]
        y = self.data[idx + self.seq_len]
        return x, y

class MultiHeadAttention(nn.Module):
    def __init__(self, d_in, d_out, dropout, num_heads, qkv_bias=False):
        super().__init__()
        assert d_out % num_heads == 0, "d_out must be divisible by num_heads"

        self.d_out = d_out
        self.num_heads = num_heads
        self.head_dim = d_out // num_heads # Reduce the projection dim to match desired output dim

        self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_key = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.out_proj = nn.Linear(d_out, d_out)  # Linear layer to combine head outputs
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        b, num_tokens, d_in = x.shape

        keys = self.W_key(x) # Shape: (b, num_tokens, d_out)
        queries = self.W_query(x)
        values = self.W_value(x)

        # We implicitly split the matrix by adding a `num_heads` dimension
        # Unroll last dim: (b, num_tokens, d_out) -> (b, num_tokens, num_heads, head_dim)
        keys = keys.view(b, num_tokens, self.num_heads, self.head_dim) 
        values = values.view(b, num_tokens, self.num_heads, self.head_dim)
        queries = queries.view(b, num_tokens, self.num_heads, self.head_dim)

        # Transpose: (b, num_tokens, num_heads, head_dim) -> (b, num_heads, num_tokens, head_dim)
        keys = keys.transpose(1, 2)
        queries = queries.transpose(1, 2)
        values = values.transpose(1, 2)

        # Compute scaled dot-product attention (aka self-attention) with a causal mask
        attn_scores = queries @ keys.transpose(2, 3)  # Dot product for each head
        
        attn_weights = torch.softmax(attn_scores / keys.shape[-1]**0.5, dim=-1)
        attn_weights = self.dropout(attn_weights)

        # Shape: (b, num_tokens, num_heads, head_dim)
        context_vec = (attn_weights @ values).transpose(1, 2) 
        
        # Combine heads, where self.d_out = self.num_heads * self.head_dim
        context_vec = context_vec.contiguous().view(b, num_tokens, self.d_out)
        context_vec = self.out_proj(context_vec) # optional projection
        return context_vec, attn_weights

# Transformer class
class Transformer(nn.Module):
    def __init__(self, num_heads, output_dim, d_in=3, d_model=64, dropout=0.1):
        super().__init__()
        self.multihead_attention = MultiHeadAttention(d_in, d_model, dropout, num_heads)
        self.regression_head = nn.Sequential(
            nn.Linear(d_model, d_model),     
            nn.LayerNorm(d_model),        
            nn.ReLU(),                       
            nn.Dropout(dropout),             
            
            nn.Linear(d_model, d_model), 
            nn.LayerNorm(d_model),       
            nn.ReLU(),                        
            nn.Dropout(dropout),               
            
            nn.Linear(d_model, output_dim) 
        )

    def forward(self, x):
        x, attn_weights = self.multihead_attention(x)
        return self.regression_head(x[:, -1, :]),  attn_weights

def evaluate_model(model, dataloader, device):
    model.eval()
    mse_loss = 0
    predictions, targets = [], []

    with torch.no_grad():
        for x_batch, y_batch in dataloader:
            x_batch, y_batch = x_batch.to(device), y_batch.to(device)
            pred, _ = model(x_batch)
            mse_loss += nn.functional.mse_loss(pred, y_batch).item()
            predictions.append(pred.cpu().numpy())
            targets.append(y_batch.cpu().numpy())

    predictions = np.concatenate(predictions)
    targets = np.concatenate(targets)
    mse = mse_loss / len(dataloader)
    corr = pearsonr(predictions.flatten(), targets.flatten())[0]  # Pearson correlation

    return mse, corr

def train_model(model, 
                train_loader, 
                val_loader, 
                criterion, 
                optimizer, 
                device,
                save_dir,
                epochs=10
                ):
    model.to(device)
    best_val_mse = float("inf")
    best_model_path = save_dir / "best_model.pth"
    print(f"Best Model Path : {best_model_path} Num Epochs : {epochs}")

    train_loss_list = []
    val_loss_list = []

    for epoch in range(epochs):
        model.train()
        train_loss = 0
        for x_batch, y_batch in train_loader:
            x_batch, y_batch = x_batch.to(device), y_batch.to(device)
            optimizer.zero_grad()
            predictions, _ = model(x_batch)
            
            if torch.isnan(predictions).any():
                print("NaN detected in predictions, stopping training.")
                return

            loss = criterion(predictions, y_batch)
            loss.backward()
            optimizer.step()
            train_loss += loss.item()

        train_loss = train_loss / len(train_loader)
        # Validation
        val_mse, val_corr = evaluate_model(model, val_loader, device)
        print(f"Epoch {epoch + 1}/{epochs}, Train Loss: {train_loss:.3f}, Validation MSE: {val_mse:.3f}, Validation Correlation: {val_corr:.3f}")

        # Save the model if it has the best validation MSE
        if val_mse < best_val_mse:
            best_val_mse = val_mse
            torch.save(model.state_dict(), best_model_path)
            print(f"New best model saved with Validation MSE: {best_val_mse}")
        
        train_loss_list.append(train_loss)
        val_loss_list.append(val_mse)

    print(f"Training complete. Best model saved with Validation MSE: {best_val_mse}")
    return train_loss_list, val_loss_list

# Test Code
def generate_future_steps(model, 
                        initial_sequence, 
                        num_steps, 
                        device):
    model.to(device)
    model.eval()

    generated_sequence = initial_sequence.clone().to(device)
    generated_sequence = generated_sequence.unsqueeze(0)  

    future_steps = []

    with torch.no_grad():
        for _ in range(num_steps):
            next_step, _ = model(generated_sequence) 
            future_steps.append(next_step.cpu().squeeze().numpy())
            generated_sequence = torch.cat((generated_sequence[:, 1:, :], next_step.unsqueeze(0)), dim=1)

    return np.array(future_steps)

def evaluate_predictions(predicted_sequence, reference_sequence):
    predicted_sequence = np.array(predicted_sequence)
    reference_sequence = np.array(reference_sequence)
    mse = np.mean((predicted_sequence - reference_sequence) ** 2)
    correlation = pearsonr(predicted_sequence.flatten(), reference_sequence.flatten())[0]

    print(f"Mean Squared Error (MSE): {mse}")
    print(f"Pearson Correlation: {correlation}")

    return mse, correlation

import matplotlib.pyplot as plt
import seaborn as sns

def visualize_attention_weights(model, input_sequence):
    # Set the model to evaluation mode and move to device if necessary
    model.eval()
    device = next(model.parameters()).device
    input_sequence = input_sequence.to(device)
    
    # Forward pass through the model to get attention weights
    with torch.no_grad():
        _, attn_weights = model(input_sequence.unsqueeze(0))  # Add batch dimension

    # Convert attention weights to numpy for plotting
    attn_weights = attn_weights.squeeze().cpu().numpy()  # Shape: (num_heads, seq_len, seq_len)

    # Plot attention weights for each head
    num_heads = attn_weights.shape[0]
    fig, axes = plt.subplots(num_heads, 1, figsize=(20, 20))
    fig.suptitle("Attention Weights for Each Head")

    for i in range(num_heads):
        sns.heatmap(attn_weights[i], ax=axes[i], cmap="viridis")
        axes[i].set_title(f"Head {i+1}")
        axes[i].set_xlabel("Key Position")
        axes[i].set_ylabel("Query Position")

    plt.tight_layout()
    plt.show()
